📖 What is Subdomain Enumeration?

Subdomain enumeration is a reconnaissance technique used in penetration testing to discover valid subdomains of a target domain. It is a critical first step in the attack surface mapping phase of security assessments.

  • Why it matters: Subdomains often host development servers, staging environments, admin panels, and APIs that may have weaker security controls than the main domain.
  • How this tool works: It queries Google's public DNS-over-HTTPS (DoH) API to resolve subdomain DNS records (A and CNAME types). No traffic is sent directly to the target — only DNS resolution is performed.
  • Common findings: Forgotten dev/staging instances, exposed admin panels, API endpoints, mail servers, cloud storage buckets, and misconfigured services.
  • Limitations: Wordlist-based enumeration only finds subdomains in the list. Real-world assessments combine this with certificate transparency logs, zone transfers, and other OSINT techniques.
